# Engineering Principles

1. One module has one responsibility.

2. Business logic never belongs in API routes.

3. Teacher Brain never reads PDFs.

4. Knowledge Platform owns educational content.

5. Student Platform owns learning state.

6. Infrastructure never contains business logic.

7. Firestore is accessed through services only.

8. Every new feature must belong to one platform.

9. APIs should orchestrate, not implement business logic.

10. Every commit must improve maintainability or student learning.
